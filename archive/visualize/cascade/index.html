The Cascade Funnel

Step 4: Exhaust. The algorithm starts at coarse resolution (d=4 bins, m=20 mass units) and refines level by level — d=4 → d=8 → d=16 → d=32 → d=64. At each level, Steps 2–3 (windowed test values + pruning) eliminate most candidates. Watch billions of configurations get tested until none remain, proving C1a ≥ 1.30.

Window Scanner

Step 2: Windowed test values. For each candidate, the algorithm checks every possible window — a contiguous range of convolution positions with a specific width ℓ. For each window, it computes a lower bound on ‖f★f‖∞. If any window exceeds the threshold c, the configuration is pruned (Step 3). Each cell below is one (ℓ, k) window check. Test a distribution in Section 2 first, then click Scan.

Refinement Tree

Step 1: Discretize, then refine. Each surviving configuration at one level splits its d bins into 2d finer bins at the next level. The parent [8, 1, 1, 10] at d=4 spawns ~250 d=8 children — about half get pruned, giving a genuine red/green mix. Click a surviving ● node to expand it and see its children at the next resolution. 5

Closing the Gap

The Sidon autocorrelation constant C1a has been bounded from above and below for decades. Our cascade algorithm pushed the lower bound from 1.28 to 1.40, narrowing the gap with the best known upper bound of 1.5029.
